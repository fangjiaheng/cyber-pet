# Q宠宝贝

基于 Electron + React + Ruffle 的桌面宠物原型项目，致敬经典 QQ 宠物。

当前版本仍使用原版 QQ 宠物相关 SWF 资源做交互与动画验证，因此只适合本地学习、原型验证和技术研究，不适合直接公开分发或商业化。若要长期发布，必须先完成角色、美术、命名、音效与文案的原创化替换。

## 当前能力

- 桌面宠物常驻显示，支持拖拽、悬浮、右键菜单和透明区域穿透
- Ruffle 常驻加载 `player.swf`，通过 `setid()` + `loadlists()` 切换动作，减少整屏闪烁
- 功能区支持动画、日常、交互、任务等入口，支持 click 和 hover 触发下拉菜单
- 下拉菜单采用原版风格：小圆点 + 原版图标（日常/清洁/治病/学习/打工/旅行）
- 喂食与清洁使用横向滚动条组件，任务使用下拉面板
- 气泡提示显示在宠物头部上方，显示气泡时不再引发宠物窗口位移
- AI 聊天窗口、AI 配置面板、本地聊天记录与设置持久化
- 完整养成系统：动态属性上限、渐进等级经验表、成长阶段、疾病框架
- 任务系统已接入”登录送礼 / 在线送礼”，支持本地持久化与重置

## 任务系统说明

- 入口位于 `任务`
- 登录送礼按周期发放，共 12 格
- 在线送礼按在线分钟数解锁，当前阈值为 10 / 40 / 70 / 100 / 130 / 160 / 190 / 220 分钟
- 每天早上 `06:00` 刷新在线送礼周期
- 送礼奖励当前映射到现有属性系统：经验、元宝、饥饿、清洁、心情、精力
- 任务状态保存在本地 `electron-store`

### 动画系统

- 使用原版 SWF 资源作为当前验证素材
- 已接入原版 102 号企鹅的多段动作 playlist 映射，常见长动作可完整播放
- 启动时会播放进场动画，再自动回到待机
- 内置动画菜单，当前主界面可浏览 8 个分类的动作入口
- 支持开发态 `player.swf` 验证面板，用来检查控制桥是否可用

### 桌面交互

- 右键菜单会贴近宠物右下方展开，点击外部区域可关闭
- 喂食和清洁使用横向滚动选择组件，动画菜单和互动菜单使用统一的可复用下拉组件
- 喂食和清洁横向滚动条会跟随功能栏居中对齐，显示在功能区正下方
- 下拉菜单已去掉横向滚动条，避免视觉干扰
- 功能栏下方不再额外显示等级、经验、元宝、心情摘要，主界面更干净
- 动画播放时自动阻止鼠标凝视打断，保证动作完整播放

### AI 助手

- 内置聊天窗口，支持流式回复
- 支持本地保存聊天历史
- `AI 助手配置` 面板支持配置 Base URL、API Key、默认模型
- 支持测试连接和保存后即时生效
- 设置面板和聊天窗口都支持拖动

### 养成与成长系统

- 喂食、清洁、玩耍、休息等基础互动
- 喂食提供 10 种食物、清洁提供 6 种用品，通过横向滚动条选择
- **渐进等级系统**：400 级经验表，替代原来的固定经验/级
- **动态属性上限**：饥饿/清洁 = 3000 + 100×min(level,30)，心情上限 1000
- **属性衰减**：饥饿/清洁 -5~8/分，心情 -2~4/分，mood<600 额外 -2
- **成长阶段**：蛋(≤4级)、幼年(5-8级)、成年(≥9级)，动画按阶段动态加载
- **心情外观**：成年阶段根据心情切换 happy/peaceful/upset/sad 四种外观
- **疾病系统框架**：3 条疾病链、不适计数器、健康 0-5 级
- **存档迁移**：旧版数据自动缩放到新属性范围
- 宠物状态会自动持久化

## 技术方案

### 动画播放

当前播放链路不再为每个动作单独重建播放器，而是：

1. 常驻加载 `public/player.swf`
2. 使用 `setid()` 指定角色
3. 使用 `loadlists()` 切换播放列表
4. 在 `src/renderer/utils/swfPlaylist.ts` 中做业务动作到原始 playlist 的映射
5. 对部分多段动作保留原版 playlist，避免只播放第一段

### 窗口布局

- `pet` 与 `bubble` 模式统一使用相同的锚点计算
- 气泡窗口尺寸与宠物窗口一致，避免切换时出现视觉跳位
- 右键菜单与功能区下拉菜单使用独立尺寸，但都基于统一锚点进行定位
- 主窗口是透明无边框 Electron 窗口，默认尺寸 `360x420`
- 主进程通过 `setIgnoreMouseEvents` 控制穿透
- 渲染层通过 `resizeWindow` 在宠物、右键菜单、下拉菜单、聊天、设置之间切换窗口尺寸
- `main.tsx` 已移除 `StrictMode`，避免开发环境下 Ruffle 因双挂载出现异常

### 成长与状态

- 成长配置在 `src/renderer/stores/growthConfig.ts`（400 级经验表、动态属性上限、阶段判断）
- 成长引擎在 `src/renderer/stores/growthEngine.ts`（每分钟成长值、衰减计算）
- 疾病系统在 `src/renderer/stores/diseaseSystem.ts`（3 条疾病链、健康等级）
- 存档迁移在 `src/renderer/stores/migration.ts`（旧版数据自动缩放）
- 阶段感知 SWF 路径在 `src/renderer/utils/stageSwfResolver.ts`

### 本地存储

`electron-store` 当前持久化以下内容：

- 宠物状态与养成数据（含成长阶段、疾病状态、存档版本）
- 登录送礼 / 在线送礼状态
- AI 配置
- 聊天记录
- Token 统计记录

## 测试

当前仓库包含两组基础测试：

- `tests/petWindowLayout.test.ts`
  - 验证宠物窗口与气泡窗口锚点一致，防止显示气泡时宠物位移
- `tests/taskGift.test.ts`
  - 验证登录送礼解锁与领取
  - 验证在线送礼按在线时长解锁
  - 验证 `06:00` 刷新逻辑
  - 验证登录送礼 12 天周期轮换

运行命令：

```bash
npm run type-check
npm run test
```

## 已知边界

- 当前资源仍直接依赖原版 QQ 宠物素材，不适合直接对外发布
- 只把收费改成”打赏”并不能解决版权、商标和混淆风险
- 当前任务奖励采用”映射到现有属性系统”的方式，尚未接入完整道具背包
- 疾病系统框架已就绪，但 UI（药品选择条、生病/死亡动画）尚未实现
- 阶段切换过渡动画（蛋→幼年、幼年→成年）尚未接入
- 少量动作虽然已经能完整播放，但业务层的”动作开始 / 结束 / 回待机”管理仍有进一步收敛空间
