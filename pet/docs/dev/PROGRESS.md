## 当前阻塞和风险

### 阻塞 1：顶部留白

桌宠拖到屏幕最顶端时仍有一段不可到达区域。
这个问题已经确认不是单个 CSS 常量能解释的，需要从 Electron 窗口边界和渲染布局一起排查。

### 阻塞 2：原版素材风险

当前仓库仍直接依赖原版角色和动画资源。
这意味着项目现在适合作为原型和研究基座，但不适合作为正式商业产品公开运营。

## 当前最值得做的三件事

1. 修复顶部留白问题
2. 收敛动作状态机，减少业务层定时器
3. 启动原创化替换方案，为未来公开发布做准备

## 当前阶段定位

如果继续推进，这个仓库最合理的角色是：

- 作为“原创桌面萌宠产品”的技术底座
- 保留当前动画播放、菜单、AI 配置、持久化等实现
- 逐步把依赖原版素材的部分替换掉
